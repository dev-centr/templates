= Common Template

The `_common` workspace template provides baseline tooling that every repository can rely on. Add it to any VS Code workspace when you want the shared documentation, git, and formatting helpers without copying files into your project.

== When to include it

* You are working in a multi-repo workspace and want consistent Git tooling (GitHub Pull Requests, GitLens, Git Graph).
* You need the shared documentation stack (Prettier, Prettier ESLint, Mermaid, AsciiDoc, SVG editor).
* You want YAML schema hints for `.github/FUNDING.yml` and other shared automation files.

== Included configuration

* `.vscode/extensions.json` – curated list of cross-cutting extensions. They all ship on the community-run OpenVSX marketplace, so they install cleanly in Cursor and other VS Code forks.
* `common.code-workspace` – shared YAML schema mappings and other base tweaks that are safe to load alongside any repository.

[IMPORTANT]
====
**Settings Precedence:** VS Code settings follow a hierarchy: **User > Workspace > Folder**. However, **Language-specific settings** (e.g., `[json]`) always override global settings within the same or higher priority levels. 

To ensure project standards "win" against personal user defaults (like a global Prettier preference), we use explicit language-level overrides in our workspace files. If you find your personal settings are interfering, ensure your workspace-level language overrides are correctly defined.
====

== How to use it

To use this template, open the `common.code-workspace` file in this folder and copy the `settings` and `extensions` blocks into your project's `.code-workspace` file.

[IMPORTANT]
====
You should also copy the corresponding blocks from your chosen tech stack template (e.g., `javascript.code-workspace`) and merge them into your project's workspace file to ensure complete coverage.
====

Copying these blocks directly into your project ensures that your development environment is fully configured with the shared standards of the Dev Center, without needing to reference external folders.

If you need to copy a specific configuration file (for example `.github/FUNDING.yml`), open the template folder directly and copy the file into your repo.
