= JavaScript / TypeScript Template

The `javascript` template layers JavaScript-specific tooling on top of the shared `_common` configuration. Use it whenever you need consistent formatting, linting, and workspace ergonomics for Node, browser, or hybrid TypeScript projects.

== What it provides

* Baseline JavaScript/TypeScript editor settings live in `_common` (dprint and language-specific formatter overrides). This template adds a starter `.vscode` folder you can copy into new repositories.
* Opinionated defaults that hide noisy folders like `node_modules`, `.git`, and `.next` while keeping Git status scoped to your project come from `_common` when you merge its `.vscode/settings.json`.

This template adds `xabikos.JavaScriptSnippets` on top of the shared list in `workspaces/_common`. Add more JavaScript-specific extensions in `.vscode/extensions.json` once they are available on OpenVSX.

[IMPORTANT]
====
**Settings Precedence:** When you open a single folder, VS Code applies **User** settings first, then **Folder** (`.vscode/settings.json`). **Language-specific settings** (e.g., `[json]`) override global keys at the same scope. 

To ensure project standards "win" against personal user defaults (like a global Prettier preference), we use explicit language-level overrides in `.vscode/settings.json`. If you find your personal settings are interfering, ensure your folder-level language overrides are correctly defined.
====

== How to use it

To use this template, copy `.vscode/settings.json` and `.vscode/extensions.json` from this folder into your project (merge with any existing `.vscode` files).

[IMPORTANT]
====
For a complete setup, merge in the `.vscode` files from `workspaces/_common/` as well.
====

Copying or merging these files into your project ensures that your development environment is fully configured with the shared standards of the Dev Center, without needing to reference external folders.
