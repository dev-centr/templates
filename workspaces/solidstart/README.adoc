= SolidStart Template

The `solidstart` template provides a specialized environment for building modern, full-stack applications with the SolidStart framework. It includes optimized settings for SolidJS components, Nitro server integration, and reactive development.

== Included extensions

All recommendations come from the community-run OpenVSX marketplace, ensuring compatibility with Cursor and other VS Code forks:

* `solid.solid-js` – Official SolidJS support, including syntax highlighting, IntelliSense, and snippets for reactive primitives.
* `biomejs.biome` – Fast, unified tool for formatting and linting JavaScript, TypeScript, and CSS.

== How to use it

To use this template, copy `.vscode/settings.json` and `.vscode/extensions.json` from this folder into your project (merge with any existing `.vscode` files).

[IMPORTANT]
====
For a complete setup, merge in the `.vscode` files from `workspaces/_common/` as well.
====

Including these settings adds Solid-specific snippets and optimizations while maintaining the shared standards from the `_common` template.

== Features

* **Tailored Formatting**: Pre-configured settings to work seamlessly with SolidJS's JSX-based reactivity.
* **Optimized Search**: Workspace excludes for `.solid/`, `.output/`, and `.nitro/` to keep search and the explorer focused on your source code.
* **Full-stack Ready**: Prepared for SolidStart's Nitro-based server-side capabilities.
